User Flow Summary

  • Step 1: User pauses content and backgrounds/closes the app ‚Üí System triggers notification after delay
  • Step 2: User receives lock screen notification with basic info (title, episode, progress)
  • Step 3: User expands notification to see full details (thumbnail, progress bar, actions)
  • Step 4: User taps "Resume Now" ‚Üí Deep link to video player and resume playback
  • Alternative: User taps "Later" ‚Üí Notification dismissed, re-sent after configured delay

Technical Requirements

  • Rich notification support (Android: NotificationCompat, iOS: UNNotificationContentExtension)
  • Background service to track playback state and trigger notifications
  • Deep linking configuration for "Resume Now" action
  • Local storage for resume position and timestamp
  • Analytics tracking for notification impressions, taps, and dismissals
  • Notification scheduling with configurable delay (e.g., 30 minutes after pause)
