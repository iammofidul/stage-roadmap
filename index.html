üìã Detailed Initiative Breakdown

Month 1: Foundation & Quick Wins

Initiative 1: Fix Trial Activation Tracking

Problem: trial_activated event missing from codebase - can't accurately measure conversions

Solution:

  • Add explicit trial_activated event in payment success flow
  • Track trial_type (standard/promotional), trial_duration, payment_method
  • Enable trial_events.dart (currently commented out)
  • Add trial_activation_source tracking (paywall_variant, promotion_id)

Expected Outcome: Visibility into trial conversion funnel, enable A/B test measurement

Team: 1 Developer (Backend integration experience)

Files to Modify: trial/data/services/trial_events.dart, payment/ui/screens/payment_success_screen

Initiative 2: Paywall A/B Testing Framework

Problem: 72.5% drop-off at paywall - need to test different presentations

Solution:

  • Leverage existing paywall_view_experiment_util.dart framework
  • Create 3 paywall variants: Control, Value-Focused, Urgency-Focused
  • Test: pricing presentation, CTA copy, social proof, video backgrounds
  • Implement variant tracking in Amplitude with experimentation service
  • Add paywall interaction heatmap tracking (scroll depth, element clicks)

Expected Outcome: Paywall conversion 27.5% ‚Üí 32% (+4-5pp)

Team: 2 Developers (UI + Analytics)

Dependencies: Initiative 1 (tracking must be in place first)

Initiative 3: D0 Onboarding Optimization

Problem: D0 consumption declining from 90% to 66% - first-time experience degrading

Solution:

  • Streamline trial_onboarding flow - reduce steps from 5 to 3
  • Pre-select top 3 categories based on region/demo (no manual selection)
  • Auto-play first video immediately after trial activation
  • Add "Skip to watching" CTA on tutorial tooltips
  • Implement progressive onboarding (teach features while watching)
  • Track onboarding_step_duration to find friction points

Expected Outcome: D0 consumption 66% ‚Üí 75% (+9pp)

Team: 3 Developers (2 UI, 1 Backend)

Files to Modify: trial_onboarding/logic/, trial_onboarding/ui/screens/

Initiative 4: Video Player UX Improvements

Problem: Only 55% reach 90% completion - half of viewers drop off

Solution:

  • Reduce initial buffering - optimize video start time (track consumption_start_time)
  • Add "Skip Intro" button for series content
  • Implement "Watch Next" auto-play countdown (5 seconds)
  • Add progress bars on thumbnails (show watched %, resume point)
  • Fix video quality auto-switching (track quality_change events)
  • Add "Resume watching" banner on home screen

Expected Outcome: 90% completion 55% ‚Üí 62% (+7pp)

Team: 2 Developers (Video player specialists)

Files to Modify: consumption/logic/cubit/consumption_cubit.dart, consumption/ui/

Initiative 5: Post-Payment Engagement Flow

Problem: Trial Day-1 retention only 27% - need immediate post-purchase engagement

Solution:

  • Enhance payment_success_screen with personalized recommendations
  • Add "Watch Now" CTA to top recommended content
  • Send immediate push notification with content recommendation (1 hour after payment)
  • Create "Your Trial Journey" widget showing days remaining, content to explore
  • Track post_payment_first_watch event and time_to_first_watch

Expected Outcome: Trial D1 retention 27% ‚Üí 35% (+8pp)

Team: 2 Developers (UI + Push notification)

Files to Modify: payment/ui/screens/payment_success_screen/, modules/firebase_notification.dart

Month 2: Retention & Re-engagement

Initiative 6: Dynamic Push Notifications

Problem: Only pre-scheduled notifications - no behavior-triggered re-engagement

Solution:

  • Build notification campaign manager with trigger rules
  • Implement 5 campaigns: Inactive (24h), Lapsed (D3), Half-watched reminder, New content alert, Trial expiring (D5)
  • Personalize notifications with user's watched content genres
  • Add rich media (content thumbnails, video previews)
  • Track notification_sent, notification_opened, notification_converted events
  • Implement optimal send time prediction (user activity patterns)

Expected Outcome: Trial D7 retention 3.3% ‚Üí 6% (+2.7pp)

Team: 3 Developers (2 Backend, 1 Mobile)

Dependencies: Requires backend notification service enhancement

Initiative 7: Churn Prevention Campaigns

Problem: 96.7% trial churn by Day 7 - need intervention for at-risk users

Solution:

  • Build churn risk scoring model (watch time, session frequency, engagement)
  • Create "at-risk" user segment (low engagement D2-D4)
  • Implement in-app interventions: personalized content recommendations, limited-time offers
  • Add "Why we think you'll love this" messaging for recommended content
  • Test gamification: "Complete 3 videos to unlock reward"

Expected Outcome: Trial churn 96.7% ‚Üí 90% (-6.7pp)

Team: 2 Developers (Analytics + UI)

Files to Create: features/churn_prevention/

Initiative 8: Personalized Content Recommendations

Problem: Home ‚Üí playback 64% - 36% land but don't engage within 10 minutes

Solution:

  • Enhance recommendation algorithm with collaborative filtering
  • Add "Because you watched X" sections on home screen
  • Implement "Trending in your region" row
  • Add "Quick picks for you" carousel (< 10 min content)
  • Personalize row order based on user behavior
  • Track recommendation_shown, recommendation_clicked, recommendation_watched events

Expected Outcome: Home ‚Üí playback 64% ‚Üí 72% (+8pp)

Team: 3 Developers (2 Backend ML, 1 Mobile UI)

Initiative 9: Watchlist Engagement

Problem: Low watchlist usage - not driving return visits

Solution:

  • Add prominent "Add to Watchlist" CTA on all content cards
  • Implement "Your Watchlist" home screen widget (top 3 items)
  • Send push notification when watchlist item becomes available
  • Add "Friends are watching" social proof (if account sharing active)
  • Track watchlist_add, watchlist_remove, watchlist_watch events

Expected Outcome: D0 consumption 75% ‚Üí 80% (+5pp)

Team: 2 Developers

Files to Modify: watchlist/

Initiative 10: In-App Messaging System

Problem: Limited ability to communicate with users in real-time

Solution:

  • Integrate CleverTap in-app messaging (already in codebase)
  • Create message templates: Onboarding tips, Feature announcements, Personalized offers
  • Implement message targeting by segment (trial vs paid, engagement level)
  • Add dismissible banners, modals, and tooltips
  • Track message_shown, message_clicked, message_dismissed events

Expected Outcome: +3-5% across multiple funnels

Team: 2 Developers

Goal: Optimize based on Month 1-2 A/B test results

  • Launch follow-up tests: Pricing tiers, Trial duration (7d vs 14d), Payment method order

Goal: Make content discovery more engaging and personalized

Solution:

  • Implement swipeable content cards (Tinder-style) on home screen
  • Add "Not interested" feedback to improve recommendations
  • Create "Explore" tab with category-based browsing
  • Add video previews on hover/long-press
  • Implement "Surprise me" shuffle feature

Expected Outcome: Home ‚Üí playback 72% ‚Üí 78% (+6pp)

Initiative 13: Engagement Scoring System

Goal: Quantify user engagement for targeting and analysis

Solution:

  • Build engagement score formula: watch time (40%), frequency (30%), completion rate (20%), social actions (10%)
  • Create user segments: Power users (score >80), Active (50-80), At-risk (<50)
  • Add engagement score to analytics events as user property
  • Build engagement trend dashboard (internal use)

Expected Outcome: Enable targeted campaigns, improve retention prediction

Team: 2 Developers (Analytics specialists)

Initiative 14: Retention Analytics Dashboard

Goal: Real-time visibility into retention metrics

Solution:

  • Build internal dashboard showing: Cohort retention curves, Funnel conversion rates, Churn risk by segment
  • Add alerts for metric degradation (e.g., D0 consumption drops below 70%)
  • Integrate Amplitude charts via API
  • Create weekly retention report automation

Expected Outcome: Faster identification of issues, data-driven decisions

Team: 2 Developers

  • Add social proof ("1.2M users joined via referral")
