üìù Implementation Notes

1. Entry Point: Content Title Screen (episode_details.dart)

  • Location: Small challenge badge below content title, above "Start Watching" button
  • Visibility Check: Use SubscriptionStatusUtil.isSubscriptionActive() - Only show to subscription_active users
  • Badge Content: Show progress (850K / 1M views), completion percentage (85%), and time remaining (3 days)
  • Click Action: Opens full challenge details as a bottom sheet or new screen
  • Animation: Subtle pulse effect on badge to draw attention without being intrusive

2. Challenge Configuration (Admin-Side)

  • Challenge Type: Show/Movie-specific (link challenge to specific content slug)
  • Goal Setting: Admin sets view target (e.g., 1M views) and duration (e.g., 7 days)
  • BTS Content Link: Admin uploads/links behind-the-scenes video that unlocks upon completion
  • Progress Tracking: Real-time view count aggregation for the specific show/movie

3. Behind-the-Scenes Reward System

  • Unlock Trigger: When view count >= target, automatically unlock BTS content
  • BTS Content Types: Blooper reels, cast interviews, making-of documentaries, deleted scenes
  • Access: BTS content appears as a new episode in the show's episode list
  • Notification: Push notification to all subscription_active users: "Challenge Complete! Unlock BTS content now üéâ"
  • Permanence: Once unlocked, BTS content remains accessible to all subscription_active users

4. Leaderboard Implementation

  • Ranking Logic: Rank users by number of episodes watched for that specific show/movie
  • Display: Show top 50 contributors with gold/silver/bronze badges for top 3
  • User Position: Highlight user's own ranking (e.g., #234) with green border
  • Real-time Updates: Refresh leaderboard every 5 minutes or on pull-to-refresh

5. Analytics Events

  • show_challenge_badge_viewed - Badge displayed on episode_details.dart
  • show_challenge_clicked - User clicks badge to view details
  • show_challenge_contribution_tracked - Episode view added to challenge goal
  • show_challenge_leaderboard_viewed - User views leaderboard
  • show_challenge_completed - Challenge goal reached
  • bts_content_unlocked - BTS content becomes available
  • bts_content_played - User plays BTS content

6. User Restriction (Subscription-Active Only)

  • Check Method: SubscriptionStatusUtil.isSubscriptionActive()
  • Badge Visibility: Only show challenge badge if user is subscription_active
  • Contribution Tracking: Only count views from subscription_active users toward goal
  • BTS Access: Only subscription_active users can access unlocked BTS content
  • Upgrade Prompt: Trial users see "Subscribe to join challenges" CTA instead of badge

7. Example Challenges
