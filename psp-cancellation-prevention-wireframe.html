üì± Complete User Flow - Simplified

  • Step 1: User pauses subscription (app or PSP) ‚Üí API: paymentRepository.pausePlan()
  • Step 2: Notification sent: "Trial Paused" / "Trial Revoked" + "You can not watch any STAGE content"
  • Step 3: User taps notification ‚Üí Opens FOMO journey (Popular unwatched + Upcoming shows)
  • Step 4: User tries to play Bideshi Bahu or any show ‚Üí Content blocked screen shown
  • Step 5A (App Pause): Simple "Resume" button ‚Üí Tap ‚Üí resumePlan() API ‚Üí Success popup
  • Step 5B (PSP Revoke): "Reactivate Trial" button ‚Üí Deeplink to PhonePe ‚Üí Authorize ‚Üí Return to STAGE ‚Üí Success
  • Step 6: Success screen (Hindi copy) with continue watching recommendations

üé® Design Patterns from Codebase

  • FOMO Journey: Based on motivator_popup.dart (700 lines) - carousel, video/image support, FOMO messaging
  • Resume Banner: Matches resume_subscription_banner.dart - expiry date, green CTA, "Remind Later" option
  • PSP Banner: Similar to trial_mandate_resume_psp.dart - mandate setup CTA, deeplink redirect
  • Success Popup: Identical to restart_subscription_popup.dart - Hindi copy, success GIF, green CTA
  • Dark Theme: Background: 0xFF121212 | Cards: 0xFF171717 | Borders: 0xFF3f3f3f
  • Green CTA: 0xFF248B3B (matches all dialogs in codebase)
  • Typography: Bold headings (700-900 weight), BEBEBE for secondary text, 737373 for meta

üîß Technical Implementation

  • Notification Trigger: After paymentRepository.pausePlan() success, send push with deeplink
  • FOMO Screen Data: Fetch from API/remote config: upcoming shows (next 24-48h) + trending (real-time viewer counts)
  • Content Gate: On play attempt, check subscription status ‚Üí Show blocked screen if paused/revoked
  • App Pause Flow: resumePlan() ‚Üí fetchRecurringSubscriptionStatus() ‚Üí Show restart_subscription_popup
  • PSP Flow: Check isUserEligibleForNewTrialMandateSetup ‚Üí Deeplink: stage://resume/har/hin/payment?planId=X
  • Analytics: Track notification_sent, notification_clicked, fomo_journey_shown, content_blocked_shown, resume_clicked, mandate_setup_clicked, success_shown

üéØ Why This Simple UX Works

  • Clear Notification: Direct message - "Trial Paused" + "You can not watch any STAGE content" (no ambiguity)
  • FOMO Before CTA: Shows popular unwatched + upcoming content BEFORE asking to resume
  • Simple Buttons: "Resume" (app) / "Reactivate Trial" (PSP) - clear action verbs, no confusion
  • Natural Flow: User tries to play Bideshi Bahu ‚Üí blocked ‚Üí CTA feels necessary (not pushy)

  • Matches Codebase: Uses existing motivator_popup pattern, resume API flows, success popup
  • Low Cognitive Load: Visual cards with social proof, simple CTAs, no overwhelming choices
