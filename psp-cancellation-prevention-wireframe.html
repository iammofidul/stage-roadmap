Push Notification Strategy - FOMO & Urgency

  • Variant 1 - Upcoming Release: "üò± Bideshi Bahu Season Finale Tomorrow!" - Creates urgency around must-watch content
  • Variant 2 - Trending/Social Proof: "üö® 2M+ watching Meera right now" - FOMO from missing popular content
  • Variant 3 - New Content: "‚è≥ 5 Days Left! New Episodes Just Dropped" - Countdown urgency + fresh content
  • Variant 4 - Personal Context: "üëÄ You left Episode 7 at the best part..." - Leverages incomplete viewing behavior
  • Timing: Sent 30 min, 6 hours, and 24 hours after pause/revocation - Tests optimal timing for urgency vs annoyance
  • Rich Notification: Expanded view shows multiple trending shows creating compound FOMO effect

Flow Comparison - PSP vs App

  • PSP-Side Revoked: Push notification ‚Üí Content blocked ‚Üí Re-register dialog ‚Üí Redirect to PSP ‚Üí Authorize mandate ‚Üí Return to STAGE ‚Üí Success
  • App-Side Paused: Push notification ‚Üí Content blocked ‚Üí Resume dialog ‚Üí One tap resume ‚Üí Success (no PSP redirect)
  • Key Difference: PSP requires full mandate setup (high friction) | App allows instant resume (low friction)
  • Entry Points: Push notifications + content gate when trying to watch | Notifications drive proactive re-activation

Technical Implementation

  • Content Gate: Check subscription status before playback ‚Üí Show blocked screen if paused/revoked
  • Status Detection: Distinguish PSP-revoked vs App-paused via backend status field
  • PSP Flow: Deep link to PhonePe/Paytm mandate setup ‚Üí Return URL back to STAGE ‚Üí Verify mandate ‚Üí Resume content
  • App Flow: Call resume API ‚Üí Update subscription status ‚Üí Enable content immediately
  • Personalization: Show watched content count, days remaining, continue watching list

Expected Impact

  • Notification Open Rate: 40-50% (industry benchmark for immediate post-cancellation push)
  • PSP Revoked Users: 81K/month cancellations ‚Üí 45% open rate ‚Üí 36K opens ‚Üí 20% conversion ‚Üí 7.2K recovered
  • App Paused Users: 35K/month pauses ‚Üí 50% open rate ‚Üí 17.5K opens ‚Üí 50% conversion ‚Üí 8.7K recovered
  • Content Gate Recovery: Additional 5-8K from users who encounter blocked screen without notification
  • Total Recovery: 21-24K trials/month saved (notifications drive 70% of recovery)
  • Revenue Impact: ‚Çπ10-12 Lakhs MRR from recovered trials (2,100 paid subscribers √ó ‚Çπ499 ARPU)
  • Why Notifications Work: FOMO from upcoming/trending shows + urgency from countdown + low effort CTA

Key Analytics Events

  • retention_notification_sent - Notification dispatched (variant: upcoming | trending | new_content | personal_context)
  • retention_notification_delivered - Notification successfully delivered to device
  • retention_notification_clicked - User taps notification (tracks which variant drives highest CTR)
  • content_blocked_shown - User sees blocked screen (source: psp_revoked | app_paused | notification_click)
  • reregister_mandate_clicked - PSP flow: User clicks re-register button
  • resume_subscription_clicked - App flow: User clicks resume button
  • psp_redirect_initiated - PSP flow: Redirected to PhonePe/Paytm
  • mandate_reregistered - PSP flow: New mandate successfully created
  • trial_resumed - App flow: Subscription resumed from pause
  • retention_flow_success - User successfully resumes and starts watching (tracks full funnel from notification to playback)
