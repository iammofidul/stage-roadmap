1. Overview

What is Server-Driven UI (SDUI)?

Server-Driven UI allows you to:

  • Update app UI without app store releases
  • Control screens from backend
  • A/B test layouts server-side
  • Push instant fixes and changes

Business Logic Example (Cannot change with SDUI)

From payment_view_model.dart - GST price calculation:

String getPayablePrice(SubscriptionPlan? selectedPlan, String? country) {
  if(selectedPlan == null) return '';

  final gstExpStatus = Preferences.readSyncData(Preferences.gstExpStatus);

  if(gstExpStatus == GstExperimentStatus.applied.name
     && country?.toUpperCase() == Consts.countryIndia) {
    return '$currencySymbol ${selectedPlan.payingPriceWithGst}/-';
  }
  return '$currencySymbol ${selectedPlan.payingPrice}/-';
}

Why SDUI can't change this: Experiment status check, country validation, and GST calculation must stay in code. SDUI can display the result, not modify the logic.

# Install CLI
dart pub global activate stac_cli

# Add dependencies
flutter pub add stac stac_core

# Initialize
stac init

# Build Dart → JSON
stac build

# Deploy (optional cloud)
stac deploy
